  Enrolamiento con Certificado Digital : Use Case

Flujo Normal

  1. El usuario accede al portal SAU
  2. El usuario registra CI / RUC
  3. El sistema consulta si el usuario posee o no certificado digital
  4. El usuario indica que si posee certificado digital
  5. El sistema direcciona verificación de validez del certificado a la entidad certificadora
  6. La entidad certificadora certifica la validez del certificado
  7. El usuario ingresa sus datos
  8. El usuario acepta Términos y Condiciones
  9. El sistema SAU genera formulario XML
  10. El certificado digital del usuario firma electrónicamente el formulario generado en el paso anterior
  11. El sistema SAU crea el usuario en su repositorio
  12. El sistema SAU envia email al usuario con el link de activación en el sistema
  13. El usuario ingresa al portal SAU y registra su contraseña
  14. El sistema SAU valida la información proporcionada por el usuario
  15. El usuario ingresa al sistema

 

Flujo Alterno (Usuario no posee certificado digital)

  1. El usuario accede al portal SAU
  2. El usuario registra CI / RUC
  3. El sistema consulta si el usuario posee o no certificado digital
  4. El usuario no posee certificado digital, es direccionado a la pantalla de términos y condiciones
  5. El usuario acepta los términos y condiciones
  6. El usuario valida datos
  7. El usuario imprime el formulario de registro del sistema SAU
  8. El usuario acude al canal autorizado
  9. El funcionario del canal autorizado ingresa datos del usuario (Cédula o RUC)
  10. El funcionario valida información con formulario de Ficha Única
  11. El sistema SAU genera formulario XML
  12. El sistema SAU firma electrónicamente el formulario generado en el paso anterior
  13. El sistema SAU envia email al usuario con el link de activación en el sistema
  14. El usuario ingresa al portal SAU y registra su contraseña
  15. El sistema SAU valida la información proporcionada por el usuario
  16. El usuario ingresa al sistema

Flujo Alterno (Usuario posee certificado digital)

  1. El usuario accede al portal SAU
  2. El usuario registra CI / RUC
  3. El sistema consulta si el usuario posee o no certificado digital
  4. El usuario si posee certificado digital
  5. El sistema SAU le pide al usuario ingresar los datos del certificado digital
  6. El sistema SAU envia los datos del certificado a la entidad certificadora.
  7. La entidad certificadora valida los datos y vigencia del certificado

  9. El usuario valida su información
  10. El usuario firma electrónicamente el formulario generado
  11. El sistema SAU envia email al usuario con el link de activación en el sistema
  12. El usuario ingresa al portal SAU y registra su contraseña
  13. El usuario ingresa el OTP proporcionado por el sistema
  14. El sistema SAU valida la información proporcionada por el usuario
  15. El usuario ingresa al sistema    
  Enviar Email con acceso al sistema : Use Case

Flujo Normal

  1. Sistema SAU genera email con información de enlace para activación del usuario
  2. Sistema SAU envía email al usuario

Flujo Alterno

  1. Email no puede ser generado
  2. SAU emite notificación indicando problemas técnicos
  Generar XML Ficha Única : Use Case

Flujo Normal

  1. Sistema SAU valida que información es correcta
  2. Sistema SAU genera ficha en formato XML con la información proporcionada
  3. El archivo emito es firmado digitalmente

 

Flujo Alterno

  1. Sistema SAU indica que existen errores en la información ingresada
  2. Sistema SAU emite notificación al usuario indicando los errores en la información
  Firma Electrónica : Use Case

Flujo Normal

  1. Usuario invoca al servicio de firma digital de la entidad certificadora
  2. El servicio de la entidad certificadora valida las credenciales del usuario y la vigencia de sus certificados
  3. Si la validación es correcta, la entidad certificadora retorna la firma digital solicitada

 

Flujo Alterno

  1. Usuario invoca al servicio de firma digital de la entidad certificadora
  2. El servicio de la entidad certificadora valida las credenciales del usuario y la vigencia de sus certificados
  3. Si la validación falla, la entidad certificadora retorna error indicando los motivos por los cuales el certificado no pudo ser firmado, esto es o por credenciales o por vigencia del certificado.
  Verificar información Usuario : Use Case

Flujo Normal

  1. Funcionario ingresa a sistema SAU
  2. Funcionario digita cédula / RUC del usuario
  3. Funcionario contrasta información física con formulario impreso del usuario
  4. Funcionario informa al sistema SAU que la información del usuario es correcta y real
  5. Funcionario procede a generar archivo XML con ficha única
  6. Sistema SAU firma electrónicamente el documento generado.
  7. Funcionario notifica al usuario que el proceso se ha completado exitosamente.

Flujo Alterno

  1. Funcionario ingresa a sistema SAU
  2. Funcionario digita cédula / RUC del usuario
  3. Funcionario contrasta información física con formulario impreso del usuario
  4. Funcionario informa al sistema SAU que la información del usuario es incorrecta.
  5. Sistema SAU emitira una alerta notificando posible adulteracion de identidad del usuario
  6. El canal autorizado ejecutará acciones de verificación de acuerdo a sus políticas internas establecidas para estos casos
