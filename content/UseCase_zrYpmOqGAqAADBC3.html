Use Case - Enrolamiento Presencial link

Description

Flujo Normal

  1. El usuario accede al portal SAU
  2. El usuario registra CI / RUC
  3. El sistema consulta si el usuario posee o no certificado digital
  4. El usuario no posee certificado digital, es direccionado a la pantalla de términos y condiciones
  5. El usuario acepta los términos y condiciones
  6. El usuario valida datos
  7. El usuario imprime el formulario de registro del sistema SAU
  8. El usuario acude al canal autorizado
  9. El funcionario del canal autorizado ingresa datos del usuario (Cédula o RUC)
  10. El funcionario valida información con formulario de Ficha Única
  11. El sistema SAU genera formulario XML
  12. El sistema SAU firma electrónicamente el formulario generado en el paso anterior
  13. El sistema SAU envia email al usuario con el link de activación en el sistema
  14. El usuario ingresa al portal SAU y registra su contraseña
  15. El sistema SAU valida la información proporcionada por el usuario
  16. El usuario ingresa al sistema

 

Flujo Alterno (Usuario posee certificado digital)

  1. El usuario accede al portal SAU
  2. El usuario registra CI / RUC
  3. El sistema consulta si el usuario posee o no certificado digital
  4. El usuario si posee certificado digital
  5. El sistema SAU le pide al usuario ingresar los datos del certificado digital
  6. El sistema SAU envia los datos del certificado a la entidad certificadora.
  7. La entidad certificadora valida los datos y vigencia del certificado
  8. El certificado es validado
  9. El usuario valida su información
  10. El usuario firma electrónicamente el formulario generado
  11. El sistema SAU envia email al usuario con el link de activación en el sistema
  12. El usuario ingresa al portal SAU y registra su contraseña
  13. El usuario ingresa el OTP proporcionado por el sistema
  14. El sistema SAU valida la información proporcionada por el usuario
  15. El usuario ingresa al sistema
